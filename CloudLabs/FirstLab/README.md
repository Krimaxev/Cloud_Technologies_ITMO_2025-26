# Отчёт по лабораторной работе №1

## Что было сделано
1. Прежде всего, изучена теория про AWS, Azure и кто вообще такие эти облачные провайдеры.

2. Проанализирован файл `Mapping Rules AWS team 1.csv`, содержащий агрегированные коды продуктов AWS `Product Code` и шаблоны типов использования `Usage Type`.

3. Для расшифровки неочевидных кодов использовалась официальная документация и страницы тарификации.

4. Проведена классификация по моделям: Каждому сервису был присвоен уровень абстракции:
    *   `IaaS` (Инфраструктура как услуга): Базовые ресурсы.
    *   `PaaS` (Платформа как услуга): Управляемые платформы.
    *   `SaaS` (Полноценный сервис как услуга): Готовые приложения, но этой модели в данных не было.

## Чему мы научились
*   Различать модели IaaS, PaaS и SaaS и в чем между ними разница разница.
*   Работать с сырыми данными биллинга облачного провайдера, расшифровывая технические коды.
*   Строить иерархическую сервисную модель для анализа затрат - от высокоуровневых категорий до конкретных типов использования.
*   Использовать официальную документацию AWS для поиска информации о назначении и тарификации сервисов.

## Выводы
В ходе работы была успешно создана сервисная модель для облачных ресурсов AWS. Эта модель переводит неструктурированные данные биллинга в понятную иерархию, которая показывает:
*   На какие ИТ-направления идут затраты.
*   По какой модели потребления (IaaS, PaaS или SaaS) используется каждый сервис.
*   SaaS не используется нигде, т.к. AWS - облачный провайдер, а не подрядчик.
