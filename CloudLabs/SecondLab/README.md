# Отчёт по лабораторной работе №2

## Что было сделано
* Консолидация исходных моделей. За основу взята сервисная модель, созданная в ЛР №1 для AWS.
* Каждой услуге Azure был найден логический аналог в категориальной модели AWS. Главным принципом было определение сути услуги, а не следование её названию.
* Для строк из файла Azure были заполнены все колонки классификации.

## Чему мы научились
*   Создавать и применять единую таксономию: Освоил методику расширения и применения одной сервисной модели к данным от разных провайдеров, что является основой для управления гибридной и мультиоблачной средой.
*   Сформировали целостную картину про облачных провайдерах.
*   Научились консолидировать данные о расходах из AWS и Azure.

## Выводы
В ходе работы была успешно создана единая кросс-провайдерная сервисная модель. Основным результатом является методология, позволяющая:

*  Преобразовать разрозненные счета от AWS и Azure в структурированный портфель IT-услуг.
*  Обеспечить основу для FinOps.
*  Управлять гибридной средой стратегически.
